# personal_blog

1.Архитектура приложения - onion arch
Она заключается в том,что каждый слой составляет какую-то конкретную предметную область, и каждый слой имеет друг к другу доступ 
через интерфейсы, который потом с помощью инверсии зависимостей заменяются классами.Т.о. код может тестироваться и без подключения к postgres, фронтенда и внешних апи.

2.Технологии:ORM Prisma(или TypeORM), DB Postgres, Backend: Ts + Node js(фреймворк Nest js), Auth: jwt, кукисы, bcrypt.

3.Bаckend: 
3.1 CRUD постов 
3.2 Авторизация хозяина (не регистрация) (/login)
3.3 Публичная страница с постами (/posts, /posts/:id )
3.4 Личный кабинет /dashboard

4.Frontend:
4.1 Страницы лк, постов, поста, авторизации ( шаблоны + верстка компонентов на React tsx)
4.2 Типизация пропсов, стейтов, компонентов, дженериков.
4.3 Добавить библиотеку tailwind css
4.4 Настроить API запросы к backend
4.5 Кастомные хуки и изучить другие хуки
